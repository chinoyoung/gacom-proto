## Branding & Design System

When designing or modifying any page, component, or visual element, **always consult the brand guidelines first**:

- **`BRANDING.md`** (repo root) — the written source of truth: color tokens, type scale, component snippets, spacing rules, do/don't, and the rules for both people and AI agents.
- **`/brand`** (live page at `app/brand/`) — the rendered reference: the same system as live components you can copy from.

Before writing any UI, read `BRANDING.md` to pick the right tokens (colors, type sizes, spacing, button styles, component patterns) and follow its rules. Match the existing system rather than inventing new values. If a design need genuinely doesn't fit the system, flag it explicitly rather than silently introducing a new pattern.

All branding instructions — including how AI agents should behave and how to keep the system in sync — live in `BRANDING.md`.
