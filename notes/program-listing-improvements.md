# Program Listing Improvements

## Move Key Elements Higher on the Page

### Surface Starting Price near top of listing
- Move Starting Price to a prominent position near the top of the Program Page.

### Redesign Overall Rating / Reviews display
- Increase the visual size of the Overall Rating and Review count.
- If a program has zero reviews, show "No reviews yet" (do not display "0").
- Add Review CTA or text like "Add first review".

### Display Years in Operation on listing
- Pull "Year Founded" from Provider and display it as "Years in Operation" near the top of the page.

### Show Last Updated Date on listing
- Display the Last Updated Date near the top of the Program Page.
- Can also be placed at the bottom of Quick Details section.
- Must reflect updates made by either the Client or an Internal Admin.

## Expand Pricing Breakdown Fields

### Add Program Cost Variations with Starting Price field
- Support multiple cost variations (e.g., 2 weeks, 3 weeks).
- Replace "Cost Per Week" with a Starting Price (USD) number input field (e.g., $10,000).
- Make Starting Price a required field.
- Update "No Program Costs" label in admin to read "Free"; display as "No Upfront Fees" on the frontend.

### Add Payment Terms checkboxes
- Add a multi-select checkbox group for Payment Terms.
- Options should include: Payment Plan, Scholarships, Upfront Deposit, Changes Allowed, and others as applicable.

### Add Refund Policy field
- Add a Refund Policy field to the pricing section.
- Support predefined categories plus an "Other" category.
- Allow an optional link to the provider's external refund policy page.

### Elevate Salary & Compensation section (TA, JA, IA listings)
- Move Salary & Compensation higher on Teaching Abroad (TA), Jobs Abroad (JA), and Internship Abroad (IA) listings.
- Add a Starting Hourly Rate sub-field.
- Feature a callout at the top of the listing (e.g., "This program includes compensation").

## Specify Listing Details

### Fix Age Requirement display logic
- When specific age values are set, display the actual values (e.g., "18-25").
- Only display "All Ages" when the "All Ages" option is explicitly selected.

### Add optional notation to Inclusions
- Allow individual inclusion items to be marked as optional.
- Display the optional notation clearly on the frontend listing.

### Add Exclusions field
- Create a new Exclusions field (mirroring the Inclusions field structure).
- Display on the frontend listing page.

### Add Deposit / Application Fee to Application Procedures
- Add a Deposit / Application Fee option to the Application Procedures section.
- Add input to accept a specified amount.
- Make it a tooltip in frontend.
- Link to refund policy.

### Add Program Tags
- Implement tagging support for programs (e.g., LGBTQI+, Sustainability).
- Display tags on the listing page.
- Allow tags to be managed via admin.

## Reviews

### Add Pros & Cons to Review form and display
- Add Pros and Cons fields to the Review submission form.
- Display submitted Pros & Cons on the frontend review section of the listing.
- Add Tag Confirmation.

## Inquiry Form Integrations

### Integrate Program Inquiry Form on Listing Page
- Embed the Program Inquiry Form directly on the Program Listing Page.
- Future idea (backlog): Support form integration from client's own site embedded on the listing.

### Integrate Provider Inquiry Form on Provider Page
- Embed the Provider Inquiry Form directly on the Provider Page.
